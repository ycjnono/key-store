# Key-Store 使用说明

本文档面向最终用户，说明 Key-Store 的安装、日常使用、数据备份与卸载。

---

## 1. 产品简介

Key-Store 是一款**完全离线**的账户密码管理工具，适用于：

- 服务器、数据库、网站、VPN 等账号的统一保管
- 按分类组织账户，支持搜索与快速复制
- 导出/导入 SQLite 备份，便于迁移或灾备

所有数据保存在本机，主密码用于解锁；程序退出或一段时间无操作后会自动锁定。

---

## 2. 安装

### 2.1 系统要求

- Windows 10 / 11（64 位）
- 约 200 MB 磁盘空间（含 Electron 运行时）

### 2.2 安装步骤

1. 运行 `KeyStore-1.0.3-Setup.exe`
2. 按向导选择安装目录（可自定义）
3. 安装完成后可从桌面或开始菜单启动 **Key-Store**

### 2.3 首次启动

首次运行需**设置主密码**：

- 主密码长度不少于 6 位
- 界面会显示密码强度提示
- 需输入两次以确认
- **主密码遗忘将无法恢复数据**，请妥善保管

设置完成后自动创建默认分类（如「服务器」「网站」等），并进入主界面。

---

## 3. 登录与锁定

| 场景 | 行为 |
|------|------|
| 日常启动 | 输入主密码登录 |
| 5 分钟无操作 | 自动锁定，需重新输入主密码 |
| 覆盖导入后 | 若源库主密码与本地不同，程序会锁定，需用**源库主密码**登录 |

---

## 4. 主界面功能

### 4.1 账户列表

- 左侧选择**分类**筛选账户
- 顶部**搜索框**按名称或用户名模糊查询
- **双击**某行查看账户详情（含解密后的密码）
- **右键**账户行可：复制用户名、复制密码、编辑、删除

### 4.2 新增 / 编辑账户

字段说明：

| 字段 | 说明 |
|------|------|
| 分类 | 必选 |
| 名称 | 必填，账户显示名 |
| 地址 | URL、IP 等 |
| 端口 | 可选 |
| 账号 | 登录用户名 |
| 密码 | 新增时必填；编辑时加载原密码，支持显示/隐藏 |
| 有效期 | 可选起止日期 |
| 备注 | 自由文本 |

**密码显示规则：**

- **新增：** 默认明文可输入；点击「隐藏」后以 `*` 脱敏
- **编辑：** 默认脱敏展示，点击「显示」可查看和修改明文

### 4.3 分类管理

通过工具栏「分类管理」可新增、重命名、删除分类。删除分类会级联删除其下所有账户。

### 4.4 导入 / 导出

导出文件为 **SQLite 数据库**（`.db`），包含分类、账户及加密元数据。

#### 导出

1. 打开「导入/导出」→「导出」
2. 选择「全部账户」或「按分类导出」
3. 选择保存路径并执行

#### 导入模式

| 模式 | 说明 |
|------|------|
| **覆盖导入** | 清空本地分类与账户，写入源库数据；同步 `crypto_meta`；导入后**锁定**，需用**导出端主密码**登录 |
| **追加导入** | 保留本地数据；按「分类名 + 账户名」匹配，存在则更新、不存在则新增 |

#### 跨库导入（主密码不同）

当导出库与当前库使用**不同主密码**时：

- 追加导入前需填写**源库主密码**
- 程序会用源密钥解密、再用本地密钥重新加密各账户密码
- 若未填写或密码错误，导入将失败并提示

同源库（`crypto_meta` 一致）则无需填写源库主密码。

## 5. 系统托盘

- 点击窗口标题栏**最小化**或**关闭**时，程序**不退出**，而是隐藏到系统托盘
- **双击**托盘图标恢复主窗口
- 托盘右键菜单：**显示主窗口**、**退出**
- 仅通过托盘「退出」或程序内退出操作才会真正关闭应用

---

## 6. 数据存储位置

| 内容 | 路径 |
|------|------|
| 密码库 | `%APPDATA%\keystore\data\key_store.db` |
| 应用配置与缓存 | `%APPDATA%\keystore\` |

> `package.json` 中应用名为 `keystore`，故 Electron 用户数据目录为 `keystore`（非带连字符的 `Key-Store`）。

**备份建议：** 定期通过「导出」功能备份 `.db` 文件，并单独妥善保存主密码。

---

## 7. 卸载

通过「设置 → 应用 → Key-Store → 卸载」或开始菜单卸载程序。

卸载时会询问：

| 选项 | 效果 |
|------|------|
| **是 — 完全清除** | 删除 `%APPDATA%\keystore` 等目录，密码库不可恢复 |
| **否 — 仅卸载程序** | 保留数据，重装后可继续使用原主密码 |

覆盖安装/升级时不会弹出清除数据提示，默认保留用户数据。

## 8. 常见问题

### Q：忘记主密码怎么办？

无法找回。只能使用此前导出的 `.db` 备份（仍需该备份对应的主密码）。

### Q：导入后无法登录？

覆盖导入后需使用**导出时库的主密码**。追加跨库导入不影响当前主密码。

### Q：复制密码失败？

可能主密钥未就绪（已锁定）或数据损坏。请重新登录后再试。

### Q：安装包或快捷方式图标未更新？

Windows 会缓存 exe 图标。可尝试：卸载后重装、将安装包改名后安装，或注销/重启后再查看。

### Q：是否支持 macOS / Linux？

当前构建配置仅面向 Windows NSIS 安装包；源码可在安装依赖后通过 `npm start` 在其他平台做开发调试，但未提供官方安装包。

---

## 9. 安全提示

1. 主密码强度越高越好，请勿与账户密码相同
2. 导出文件与主密码应分开保管
3. 在他人可访问的电脑上使用后，请通过托盘完全退出
4. 本工具不提供云同步；跨设备迁移请使用导出/导入
