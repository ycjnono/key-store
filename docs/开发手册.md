# Key-Store 开发手册

本文档面向开发人员，说明 JavaScript / Electron 版的架构、模块划分、本地开发流程与 Windows 打包方式。

---

## 1. 技术栈

| 层级 | 技术 |
|------|------|
| 桌面壳 | Electron 28 |
| 主进程 | Node.js（CommonJS） |
| 渲染进程 | 原生 HTML / CSS / JavaScript（无前端框架） |
| 数据库 | better-sqlite3（同步 SQLite） |
| 加密 | Node.js `crypto`（PBKDF2 + AES-256-GCM） |
| 打包 | electron-builder 24（NSIS 安装程序） |

---

## 2. 架构概览

```
┌─────────────────────────────────────────────────────────┐
│  renderer/          渲染进程（无 Node 权限）              │
│  index.html / dashboard.html + login.js / dashboard.js   │
└──────────────────────────┬──────────────────────────────┘
                           │ window.keystore (contextBridge)
┌──────────────────────────▼──────────────────────────────┐
│  electron/preload.js    IPC 桥接                        │
└──────────────────────────┬──────────────────────────────┘
                           │ ipcRenderer.invoke
┌──────────────────────────▼──────────────────────────────┐
│  electron/main.js       主进程                          │
│  · BrowserWindow / Tray                                 │
│  · IPC 注册与统一错误包装 { ok, data | error }          │
└──────────────────────────┬──────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────┐
│  app/                                                   │
│  config.js · crypto/cryptoService.js · db/database.js   │
│  services/authService.js · accountService.js · …        │
└─────────────────────────────────────────────────────────┘
```

### 2.1 安全模型

- `contextIsolation: true`，`nodeIntegration: false`
- 渲染进程仅通过 `preload.js` 暴露的 `window.keystore` 访问后端
- 主密钥仅存于主进程内存（`cryptoService`），锁定后清零
- 所有业务 IPC 经 `wrap()` 包装，统一返回 `{ ok: true, data }` 或 `{ ok: false, error }`

### 2.2 进程间 API 一览

定义见 `electron/preload.js`，主进程处理见 `electron/main.js` 中 `registerIpc()`。

| 命名空间 | 主要方法 |
|----------|----------|
| `auth` | `isFirstUse`, `setup`, `verify`, `lock`, `isUnlocked`, `passwordStrength` |
| `category` | `list`, `create`, `update`, `delete`, `countAccounts` |
| `account` | `list`, `get`, `create`, `update`, `delete`, `decryptPassword` |
| `export` | `all`, `categories` |
| `import` | `preview`, `overwrite`, `append` |
| `dialog` | `saveFile`, `openFile` |
| `clipboard` | `write` |
| `nav` | `login`, `dashboard` |

渲染端封装示例（`dashboard.js`）：

```javascript
async function api(promise) {
  const res = await promise;
  if (!res.ok) throw new Error(res.error);
  return res.data;
}
```

## 4. 数据与加密

### 4.1 存储路径

由 `app/config.js` 决定：

```
{Electron userData}/data/key_store.db
```

开发环境与安装版均为：`%APPDATA%\keystore\data\key_store.db`（`package.json` 的 `name` 为 `keystore`）。

### 4.2 数据库表

| 表 | 用途 |
|----|------|
| `crypto_meta` | 单行：salt、verification 密文、PBKDF2 迭代次数 |
| `category` | 分类 |
| `account` | 账户；`password` 列为 BLOB（加密后） |
| `import_log` | 导入记录（预留） |

表结构与历史 Java 版兼容，便于迁移。

### 4.3 加密流程

1. **设置主密码：** 生成 salt → PBKDF2 派生 32 字节密钥 → 加密验证串 `KEYSTORE_VERIFY` 写入 `crypto_meta`
2. **登录：** 同样派生密钥 → 解密 verification → 比对明文
3. **账户密码：** 使用内存中的主密钥，AES-256-GCM 加密；密文格式 `[IV 12B][密文][Tag 16B]`

常量（`config.js` / `cryptoService.js`）：

- `PBKDF2_ITERATIONS = 100_000`
- `AUTO_LOCK_MINUTES = 5`

### 4.4 导入导出要点

**导出（`exportService.js`）：**

- 复制 `crypto_meta`、`category`、`account` 到目标 `.db`
- 必须包含 `crypto_meta`，否则其他机器无法解密账户密码

**覆盖导入（`importService.importOverwrite`）：**

- 清空本地 category / account
- 写入源库 `crypto_meta` 与数据
- 导入后 `authService.lock()`，用户需用源库主密码登录

**追加导入（`importService.importAppend`）：**

- 若源库与本地 `crypto_meta` 不同，需提供 `sourcePassword`
- `createPasswordTransformer` 预派生源密钥一次，避免每条账户重复 PBKDF2
- 按「分类名 + 账户名」（不区分大小写）判断新增或更新

## 5. 本地开发

### 5.1 环境准备

1. 安装 [Node.js 18+ LTS](https://nodejs.org/)
2. 克隆项目到本地

### 5.2 安装依赖

**推荐（Windows）：**

```bat
install-deps.bat
```

脚本使用 `.npmrc.user` 指定 `registry=https://registry.npmmirror.com`，避免全局 `npm.taobao.org` 证书失效导致安装失败。

**手动：**

```bash
npm install --userconfig .npmrc.user
npm run postinstall   # 重建 better-sqlite3 等原生模块
```

### 5.3 启动开发版

```bash
npm start
```

等价于 `electron .`，入口为 `electron/main.js`。

### 5.4 开发注意事项

| 项 | 说明 |
|----|------|
| 热重载 | 无内置 HMR；改代码后需重启 `npm start` |
| 原生模块 | 升级 Electron 后需执行 `npm run postinstall` |
| 数据隔离 | 开发版与安装版共用 `%APPDATA%\keystore`（同一 `name`） |
| 托盘 | Windows 优先加载 `tray-icon-16.png` / `tray-icon.ico` |

## 6. 构建与发布

### 6.1 一键打包

```bat
build-exe.bat
```

步骤：

1. `install-deps.bat` — 安装依赖
2. `npm run prepare:icons` — 从 `assets/logo.png` 生成 `build/icon.ico`
3. `npm run clean:dist` — 清理 `release/`（及尝试清理 `dist/`）
4. `npm run postinstall` — 原生模块
5. `npm run build` — 调用 `scripts/build-win.js`

### 6.2 构建输出

| 产物 | 路径 |
|------|------|
| NSIS 安装程序 | `release/KeyStore-1.0.0-Setup.exe` |
| 块映射（可选） | `release/KeyStore-1.0.0-Setup.exe.blockmap` |

`npm run build:dir` 可输出未打包目录，便于调试 exe 图标等问题。

### 6.3 为何在临时目录构建

`scripts/build-win.js` 将 electron-builder 输出写到系统临时目录（`%TEMP%\keystore-electron-build`），再将 `.exe` 复制到 `release/`。

原因：IDE（如 Cursor）可能占用工作区 `dist/` 文件句柄，导致打包失败。

### 6.4 electron-builder 配置摘要

见 `package.json` → `build`：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `appId` | `com.changjiang.keystore` | 与 `setAppUserModelId` 一致 |
| `productName` | `Key-Store` | 安装目录与快捷方式名称 |
| `directories.output` | `release` | 最终输出 |
| `directories.buildResources` | `build` | 图标、NSIS 扩展 |
| `icon` / `win.icon` | `build/icon.ico` | 应用 exe 图标 |
| `nsis.installerIcon` | `build/icon.ico` | 安装程序图标 |
| `nsis.include` | `build/uninstaller.nsh` | 卸载时可选删除用户数据 |
| `win.signAndEditExecutable` | `true` | 将图标写入 exe 资源 |

### 6.5 图标生成

**推荐流程（使用项目 logo）：**

1. 将透明背景 PNG 放到 `assets/logo.png`
2. 安装 Python 依赖：`pip install Pillow`
3. 执行：

```bash
python scripts/process-logo.py
# 或构建时自动：npm run prepare:icons
```

生成文件：

- `assets/app-icon.png` / `app-icon.ico` — 应用与窗口
- `assets/tray-icon*.png` / `tray-icon.ico` — 系统托盘
- `build/icon.ico` — 打包用（含 256×256 等多尺寸）

**无 Python 时：** 可运行 `node scripts/generate-icons.js` 生成蓝色钥匙占位图标（非 logo）。

> ICO 必须包含 **256×256** 尺寸，否则 electron-builder 可能回退为默认 Electron 图标。

### 6.6 版本号与产物名

修改 `package.json` 的 `version` 后重新打包，安装包名为 `KeyStore-{version}-Setup.exe`（由 `artifactName` 模板决定）。

## 8. 主进程行为摘要

### 8.1 窗口与托盘

- 最小化 / 关闭窗口 → `hideToTray()`，不退出进程
- 托盘双击 → `showMainWindow()`
- 真正退出：托盘菜单「退出」或 IPC `window:close`（设置 `isQuitting = true`）
- 退出前：`authService.lock()`、`closeDb()`、销毁托盘

### 8.2 页面导航

- 登录成功 → `nav:dashboard` 加载 `dashboard.html`
- 锁定后 → `nav:login` 加载 `index.html`

## 9. 排错指南

### 9.1 依赖安装

| 现象 | 处理 |
|------|------|
| `npm.taobao.org` 证书错误 | 使用 `install-deps.bat` 或 `--userconfig .npmrc.user` |
| `node-gyp` / 编译 better-sqlite3 失败 | 确认网络可下载预编译包；或安装 VS Build Tools（C++ 桌面开发） |
| 大量 `Unknown user config` 警告 | 全局 `~/.npmrc` 旧镜像项，可忽略或清理 |

### 9.2 打包

| 现象 | 处理 |
|------|------|
| `dist/` 无法删除 | 关闭 Key-Store / Cursor 占用；`build-win.js` 已改用临时目录 |
| 安装包图标不对 | 确认 `assets/logo.png` 存在；安装 Pillow；执行 `npm run prepare:icons` |
| 图标仍显示旧图 | Windows 图标缓存；换安装包文件名或重启资源管理器 |

### 9.3 运行时

| 现象 | 处理 |
|------|------|
| 导入后密码无法解密 | 确认导出含 `crypto_meta`；跨库追加需源库主密码 |
| 覆盖导入后无法登录 | 使用导出端主密码，非导入前本地密码 |
| `getMasterKey is not a function` | 确认 `cryptoService` 已导出 `getMasterKey` |

## 10. 扩展开发建议

### 10.1 新增 IPC 接口

1. 在 `app/services/` 实现业务逻辑
2. 在 `electron/main.js` 的 `registerIpc()` 注册 `ipcMain.handle`
3. 在 `electron/preload.js` 的 `contextBridge` 中暴露方法
4. 渲染进程通过 `api(window.keystore.xxx.yyy(...))` 调用

### 10.2 修改数据库结构

在 `app/db/database.js` 的 `initSchema` 中追加 `CREATE TABLE IF NOT EXISTS` 或迁移逻辑；注意与 Java 版/旧备份的兼容性。

### 10.3 修改加密参数

若调整 `PBKDF2_ITERATIONS` 或算法，需考虑旧库迁移；当前实现与 Java 版保持一致以便 `.db` 互通。

---

## 11. 与 Java 版的关系

本仓库已由 Java/JavaFX 重构为 Electron 实现，核心约定保持一致：

- 数据库文件路径与表结构
- PBKDF2 + AES-GCM 加密格式
- 导出 `.db` 可在同算法版本间迁移

旧 `src/main/java` 代码已移除，仅以本 JS 工程为准进行维护与发布。
